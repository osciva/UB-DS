Acceptance Criteria 2.1: Login Soci: Contrasenya erronia

El sistema dona opció de login però la contrassenya donada és erronia.

Example del test d'acceptació 2.1
Posem un usuari existent amb contrassenya erronia: ajaleo@gmail.com amb contrassenya INVENTADA11 el sistema hauria de donar Contrassenya incorrecta.

Acceptance Criteria 2.2: Login Soci: Correu erroni

El sistema dona opció de login però el correu no coincideix amb el de cap compte.

Example del test d'acceptació 2.2
Posem un usuari existent amb correu inexistent: gatigos@gmail.com amb contrassenya INVENTADA11 el sistema hauria de donar Correu inexistent.

Acceptance Criteria 2.3: Login Soci: Contrassenya oblidada

El sistema retornarà la contrassenya d'un usuari si aquest escriu el seu correu electrònic correctament (això és només un exemple, cibersecurity people dislikes this).

Example del test d'acceptació 2.3
Posem el correu d'un usuari: ajaleo@gmail.com el sistema hauria de donar ajaleoPassw7.

Acceptance Criteria 2.4: Login Soci: Logins correctes

El sistema dona opció de login i les dades introduïdes són correctes.
